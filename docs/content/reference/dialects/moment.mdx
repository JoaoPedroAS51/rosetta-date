# `moment`

The Moment.js-style token grammar. Imported from `rosetta-date/dialects`:

```ts
import { moment } from 'rosetta-date/dialects'
```

**Spec:** [Moment.js format tokens](https://momentjs.com/docs/#/displaying/format/). This is the
grammar; the `momentjs` [library](/reference/libraries/momentjs) is its reference renderer.

## Literals

- Text is wrapped in brackets: `[...]`.
- The closing bracket `]` has **no in-band escape**, so a literal `]` is emitted between bracketed
  spans (e.g. `a]b` → `[a]][b]`). The text still round-trips intact.

See [Literals & Adjacency](/guides/literals-and-adjacency) for the cross-dialect rules.

## Aliases

Extra spellings that are **parsed** but normalize to the primary token above when rendered:

- `Y` → calendar year (like `YYYY`).
- `NN` / `NNN` → abbreviated era (like `N`).
- `a` → AM/PM (like `A`).
- `zz` → time-zone name (like `z`).

## Caveats

<Callout type="warning">
  - Narrow era (`NNNNN`) renders the **abbreviation** (e.g. `AD`), not a true one-character form.
  - Weekday number `d` is `0`–`6` (locale-dependent). Dialects that also expose a locale weekday
    number may number the days differently, so the numeric value is not guaranteed to match across
    dialects.
  - `A` and `a` both mean AM/PM — moment encodes **case**, not width — so the case distinction is not
    preserved when converting to a dialect that distinguishes by width instead.
</Callout>
