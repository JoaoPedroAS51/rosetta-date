# `strftime`

The C / POSIX `strftime` token grammar ‚Äî the `%`-directives shared by C, Python, Ruby, and PHP.
Imported from `rosetta-date/dialects`:

```ts
import { convert } from 'rosetta-date'
import { moment, strftime } from 'rosetta-date/dialects'

convert('%Y-%m-%d', { from: strftime, to: moment }) // 'YYYY-MM-DD'
convert('DD/MM/YYYY', { from: moment, to: strftime }) // '%d/%m/%Y'
```

- **Spec:** [POSIX `strftime`](https://pubs.opengroup.org/onlinepubs/9699919799/functions/strftime.html)
- **Family:** [`directive`](/reference/dialects#syntax-families) ‚Äî every token is introduced by `%`
- **Literals:** any non-directive text is literal verbatim; a literal `%` is `%%` ‚Äî see [Literals & Adjacency](/guides/literals-and-adjacency)

## A directive grammar

`strftime` belongs to the **directive** family ‚Äî the mirror image of the [**delimited**
family](/reference/dialects#syntax-families): every token is introduced by the `%` marker, and *all
other text ‚Äî letters included ‚Äî is literal*. So `Y` on its own is the literal letter `Y`; only `%Y` is
the year. Because each directive is self-delimiting, adjacent directives never need a separator
(`%H%M` is unambiguous).

## Composite directives

A handful of directives are **composites** ‚Äî a single spelling that stands for a whole sub-pattern.
They expand on parse, so converting one out yields the expanded fields:

| Composite | Expands to |
| --------- | ---------- |
| `%T` | `%H:%M:%S` |
| `%R` | `%H:%M` |
| `%F` | `%Y-%m-%d` |
| `%D` | `%m/%d/%y` |
| `%r` | `%I:%M:%S %p` |
| `%n` | newline (`\n`) |
| `%t` | tab (`\t`) |

A composite is a parse-time macro, so **rendering produces its expansion, never the composite**:
`%T` round-trips to `%H:%M:%S` (identical output, normalized spelling), the same way an alias
normalizes ‚Äî just across several tokens. `%n` and `%t` are the same mechanism applied to literal
whitespace: they expand to a literal newline/tab, so they normalize to that raw character (the output
is identical, only the `%n`/`%t` spelling is not preserved).

## Aliases

`%h` is parsed as an abbreviated month, the same as `%b`; it normalizes to `%b` when rendered.

## Caveats

<Callout type="warning">
  - **`%e` / `%k` / `%l` are blank-padded.** No other built-in dialect has a blank-padded form, so
    converting one to a dialect that lacks it has no clean target ‚Äî it falls to the unsupported-token
    policy. A dialect that does define the blank-padded form round-trips cleanly.
  - **`%C` (century) has no built-in counterpart.** No other built-in dialect renders a century, so
    converting `%C` out has no clean target ‚Äî it falls to the unsupported-token policy. A dialect
    that defines the century round-trips cleanly.
  - **The locale presets (`%c` / `%x` / `%X`) map at a fixed width.** POSIX leaves their exact form to
    the locale, so the date/time-style mapping is a reasonable approximation, not an exact one.
</Callout>
