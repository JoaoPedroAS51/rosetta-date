# Token Mapping

This is the **grammar layer** — how each dialect spells every [canonical
field](/reference/canonical-vocabulary). Each row is one canonical field; each column is a dialect.
This is the side-by-side comparison view; for one dialect's full grammar, literals, aliases, and
caveats, see its page under [Dialects](/reference/dialects).

<Callout type="info">
  A **—** means the dialect has no token for that field. Converting that field *into* the dialect
  has no clean target, so it is routed through [`onUnsupportedToken`](/guides/unsupported-tokens)
  (default: escaped as a literal). Adding a dialect adds a column — no row is pairwise.
</Callout>

Behaviour specific to a concrete tool (e.g. extensions, unrenderable tokens, option-gated tokens)
lives in [Libraries](/reference/libraries); the tokens below are the dialect grammars themselves.

<Callout type="info">
  The `ldml` dialect itself defines no Unix-epoch tokens, ISO week fields, or ordinals — those are
  library extensions (e.g. date-fns adds `t` / `T` for the epoch), not part of the bare dialect, so a
  `dialect → dialect` conversion literalizes them. See [Libraries](/reference/libraries).
</Callout>

## Localized presets

These render per the library's loaded locale and map **preset → preset**. Not every dialect defines
them: where a column shows `—` here, the presets are supplied by a *library* layered on that dialect
(e.g. `dateFns` over `ldml`), not by the bare dialect itself. See
[Libraries → Localized presets](/reference/libraries#localized-presets).

| ------- | -------- | ------ | ---------- |
| Date, short | `L` | — | `%x` |
| Date, medium | `ll` | — | — |
| Date, long | `LL` | — | — |
| Time, short | `LT` | — | — |
| Time, with seconds | `LTS` | — | `%X` |
| Date + time, medium | `lll` | — | — |
